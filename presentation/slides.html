Identifying Early Engagement Drivers

to Boost Trial Conversion

Data-Driven Insights for B2C Growth

A Take-Home Assignment by Thomas Couzineau for Dashlane

The Challenge & The Goal

Business Goal

Boost B2C revenue by improving the user onboarding flow.

Our Mission

Collaborate with Product & Marketing to increase trial conversion.

Data Goal

Identify key retention drivers in a user's first 7 days.

Key Metric & Baseline

30% 4-Week Retention

Key Questions Investigated

Which user behaviors in the first 7 days have the biggest impact on 4-week retention?

Which sign-up platforms are most (and least) successful?

How do frequency (#days) and volume (#events) compare as drivers?

Summary of Findings

Habit is King

Repeatedly logging in (4+ days) is the single strongest driver of retention.

Web Onboarding Wins

Users signing up on web are 2x more likely to be retained.

Investing in the vault matters

Adding passwords, documents and payment methods are powerful, predictive actions.

Insight 1: Habitual login is the strongest driver.

The Data

80%+ Retention

...for users who log in on 4+ different days in their first week (vs. 30% avg).

The Model

2x Odds

...of retention for every additional day a user logs in.

Insight 2: The web platform is a massive outlier.

Web Users Convert at 58%

Compared to just 16% for Android users.

2x More Likely to Retain

The model shows 'web' signup is a powerful predictor.

The 'saex' Opportunity

Largest user volume but a low 30% retention. Fixing this is a huge win.

Insight 3: "Building the vault" is a key predictor.

Top "Vault-Building" Drivers (Odds Ratio)

+93% Per additional day added new password
+44% If payment method is added
+28% If personal document is added

The aha moment isn't just using autofill, it's investing in the vault.

Actionable Recommendations: Marketing

1. Double-down on web users

Focus acquisition spend on campaigns that lead to web sign-ups.

Metric to Follow

#New users created from web

2. Target the "Vault-Builder"

Launch campaigns for users who need to store documents or payment methods, not just passwords.

Metric to Follow

%New users adding doc/payment in W1

Actionable Recommendations: Product

1. Fix non-web Onboarding

Prioritize deep-dives of the 'android' (lowest retention) and 'saex' (highest volume) onboarding flows. What does 'web' do right that they don't?

Metric to Follow

%Retention rate for Android and saex new users

Actionable Recommendations: Product

2. Drive Habit

The first 7 days are a race to build a habit.

  • Gamify it: "You're on a 2-day streak!"
  • Trigger prompts on Day 3/4 to add more passwords
Metric to Follow

% New users logging in 4+ days in W1

3. Guide Users to "Invest"

Actively prompt users to add documents and payment methods during onboarding, not just passwords.

Metric to Follow

%New users adding doc/payment in W1

Next Steps

Appendix: Feature Engineering

Key features selected for the retention model.

Feature Description
User Creation Platform The app platform used to create the user account (e.g., ios, android, web, saex).
User has event in W1 Boolean feature indicating if the user triggered a specific event at least once in the first 7 days.
(This feature exists for all event names).
#Distinct days with event in W1 Total number of distinct days the user triggered a specific event in the first 7 days.
(This feature exists for all event names).
Total #Events in W1 The total count of times a user triggered a specific event in the first 7 days.
(This feature exists for all event names).

Another candidate feature considered was "Number of days to first event".
This was ultimately not selected as it was highly correlated with the "#Distinct days with event" features.
